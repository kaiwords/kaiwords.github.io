1. What is Tailwind CSS?

Tailwind CSS is a utility-first CSS framework that helps you build modern user interfaces without leaving your HTML.

Key Features:

  • ✓ Utility-First: Pre-built utility classes for styling
  • ✓ Responsive: Built-in responsive design patterns
  • ✓ Customizable: Easy theme customization
  • ✓ Production-Ready: PurgeCSS for smaller file sizes
  • ✓ Low-Level: No opinionated component patterns

vs Traditional CSS:

❌ Traditional CSS

.btn-primary {
  background: blue;
  color: white;
  padding: 8px 16px;
  border-radius: 4px;
}

✓ Tailwind CSS

<button class="bg-blue-500 
  text-white px-4 py-2 
  rounded">
  Click me
</button>

2. Installation & Setup

Using NPM:

npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init -p

CDN (Quick Start):

<script src="https://cdn.tailwindcss.com"></script>

Configuration (tailwind.config.js):

module.exports = {
  content: ["./src/**/*.{html,js}"],
  theme: {
    extend: {
      colors: {
        primary: '#3B82F6',
      }
    }
  },
  plugins: []
};

5. Responsive Design

Tailwind uses mobile-first breakpoints. Prefix utilities with breakpoint names:

Breakpoint Prefix Min Width CSS
Mobile - 0px @media (max-width: 640px)
Small sm: 640px @media (min-width: 640px)
Medium md: 768px @media (min-width: 768px)
Large lg: 1024px @media (min-width: 1024px)
Extra Large xl: 1280px @media (min-width: 1280px)

Example:

<div class="w-full md:w-1/2 lg:w-1/3">
  Full width on mobile
  50% on tablets
  33% on desktop
</div>

6. Component Building

Example: Button Component

<!-- Basic Button -->
<button class="px-4 py-2 bg-blue-500 text-white 
  rounded-lg hover:bg-blue-600 
  transition duration-200">
  Click me
</button>

Example: Card Component

<div class="bg-white rounded-lg shadow-md p-6 
  max-w-sm mx-auto">
  <h3 class="text-lg font-bold text-gray-800 mb-2">Title</h3>
  <p class="text-gray-600 text-sm mb-4">Description</p>
  <button class="w-full bg-blue-500 text-white py-2 
    rounded-lg hover:bg-blue-600">
    Action
  </button>
</div>

Using @apply for Reusable Styles

/* In your CSS file */
@layer components {
  .btn-primary {
    @apply px-4 py-2 bg-blue-500 text-white 
           rounded-lg hover:bg-blue-600 
           transition duration-200;
  }
}

/* In HTML */
<button class="btn-primary">Click me</button>

7. Best Practices

✓ DO

  • • Use utility classes for rapid prototyping
  • • Extract repeated patterns with @apply or components
  • • Organize classes with meaningful line breaks
  • • Use responsive prefixes consistently
  • • Leverage the Tailwind plugin ecosystem

✗ DON'T

  • • Don't overuse arbitrary values (use config instead)
  • • Don't nest Tailwind classes too deeply
  • • Don't ignore the spacing scale (use consistent values)
  • • Don't forget PurgeCSS in production
  • • Don't mix utility classes with custom CSS poorly

📌 Pro Tips

  • • Use VS Code extension for autocomplete
  • • Create custom colors in tailwind.config.js
  • • Use opacity modifiers: bg-blue-500/50
  • • Combine with dark mode: dark:bg-gray-900
  • • Use Tailwind UI or Headless UI for components

📍 Position Property

Changes how element positioning works:

Positioned relative to normal position

Removed from flow, relative to parent
